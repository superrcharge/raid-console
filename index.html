⚔️ Raid Reminder Console

Friendly settings editor for the RaidHelperReminder bot. Every save is a commit to config.json — which is the deployment.

Connect to GitHub

This page talks directly from your browser to GitHub's API. It needs a fine-grained personal access token — a limited key that can only touch the one settings repository. The token is stored only in this browser (localStorage) and is only ever sent to api.github.com.

One-time token creation (about 2 minutes):

  1. Open github.com/settings/personal-access-tokens/new (log in with the GitHub account that owns the settings repo).
  2. Token name: raid-console. Expiration: pick the longest offered (you'll paste a fresh one when it expires).
  3. Under Repository access choose Only select repositories → pick RaidHelperReminder.
  4. Under Permissions → Repository permissions set exactly two: Contents → Read and write (edit settings) and Actions → Read and write (trigger test runs). Leave everything else "No access".
  5. Click Generate token (green button, bottom) and copy the github_pat_… value it shows.

Repo not in the "Only select repositories" list? Fine-grained tokens only cover repos you own. If you're a collaborator on someone else's repo (e.g. before an ownership transfer), use a classic token instead: github.com/settings/tokens/new → note: raid-console → tick the repo scope → Generate → paste the ghp_… value below (and set Repo owner to the account that owns the repo). Once the repo is transferred to you, revoke the classic token and switch to a fine-grained one.

who is expected at raids
A team = a name + who's in it (Discord role IDs, or explicit user IDs). default is used for events whose channel matches no routing rule below — don't delete it. Renaming a team updates the routing rules automatically.
